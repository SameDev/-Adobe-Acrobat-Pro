Documentação da API

INTRODUÇÃO

Essa API é uma api de um ambiente real de um TCC (Trabalho de Conclusão de Curso) do Instituto federal catarinense. Para saber mais acesse esse link: github.

Você usuário que está acessando esse site não tem permissão para fazer Requisições a esta API!

Requisições HTTP permitidas:

POST: Criar e Atualizar recursos.

GET: Listar recursos.

DELETE: Deletar recursos.

Respostas HTTP:

  • 200 OK – a solicitação foi bem-sucedida (algumas chamadas de API podem retornar 201).
  • 201 Created – a solicitação foi bem-sucedida e um recurso foi criado.
  • 400 Bad Request - a solicitação não pôde ser compreendida ou faltavam parâmetros necessários.
  • 401 Unauthorized – falha na autenticação ou o usuário não tem permissões para a operação solicitada.
  • 404 Not Found – o recurso não foi encontrado.

REFERÊNCIAS

Usuário

São ouvintes e artistas que utilizaram do login e afins.

Atributos do Usuário:

  • id Int : indentificador único
  • email String : email único do úsuario
  • nome String : nome do úsuario
  • cargo String : cargo do úsuario (ADMIN, ARTISTA e USUARIO)
  • foto_perfil String : url da imagem de perfil do úsuario
  • data_nasc DateTime : data de nascimento do úsuario

Usuário

Aqui terão instruções de único úsuario de como trabalhar com ele:

Criar Usuário

POST /user/register

Request

HEADERS

Content-Type: application/json

BODY

{
"nome": "Alisson",
"email": "alissongslaoqslaoq@gmail.com",
"senha": "1234567",
"data_nasc": "2010-12-24T00:00:00.000Z"
}

Response

Status Code: 201

HEADERS

Content-Type: application/json

BODY

"Usuário criado com sucesso"
Fazer Login

POST /user/login

Request

HEADERS

Content-Type: application/json

BODY

{
"email": "conradtsamuel11@gmail.com",
"senha": "1234567"
}

Response

Status Code: 200

HEADERS

Content-Type: application/json

Authorization: jwt token

BODY

{
 "user": {
  "id": 3,
  "email": "conradtsamuel11@gmail.com",
  "nome": "Samuel Conradt",
   "cargo": "ARTISTA",
  "foto_perfil": "https://i.imgur.com/Lr91wyf.jpg",
  "data_nasc": "2006-11-16T00:00:00.000Z"
 }
}

O JWT Token é para fazer autenticações. Passe ele em Requisições que solicite ele, ele é gerado através do login.

Atualizar Úsuario

POST /user/update/:id

Request

HEADERS

Content-Type: application/json

Authorization: JWT Token aqui

BODY

{
"nome": "Samuel Conradt",
"email": "conradtsamuel@gmail.com",
"cargo": "ADMIN",
"data_nasc": "2006-12-16T00:00:00.000Z"

Música

São músicas do sistema.

Atributos da Música:

  • id Int : indentificador único
  • nome String : mome da música.
  • artista String : nome do úsuario artista.
  • userId String : ID do úsuario que enviou a música.
  • url String : url da música
  • imageUrl String : url da foto da música
  • duracao String : duração total da música.
  • tags Array : tags da música.
  • idArtista Array : ids dos artistas da música.
